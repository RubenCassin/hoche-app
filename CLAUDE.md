# HOCHE — mémoire projet

App de **fléchettes pour bar/soirée** (React Native + Expo). **Renommée OCHE → HOCHE** (juin 2026) : le rebrand couvre l'UI/app.json/notifs ; les identifiants internes restent `Oche*`/`oche.*` (composants `OcheText`…, clés de stockage `oche.token`…, fichier `oche-db.json`) — ne pas les renommer (perte de sessions/records sinon). Testée via **Expo Go** sur un PC pendant les soirées (pas encore de build EAS). UI en **français**. Identité visuelle « broadcast / fight card » : coins carrés, fond noyer, accents ambre/brique/orange.

## Lancer / tester
- **Client** : `npx expo start` (Expo Go). Installs : toujours `npm install <pkg> --legacy-peer-deps`. Versions Expo épinglées via `node_modules/expo/bundledNativeModules.json`.
- **Backend** : `node backend/server.js` (port 3001 par défaut, `PORT` configurable). Le client trouve l'API via l'IP LAN de Metro ; override avec **`EXPO_PUBLIC_API_URL`** (HTTP **et** WebSocket en dérivent — voir `constants/theme.ts` `resolveApiBaseUrl` + `services/api.ts` `socketUrl`).
- **Migrations Prisma** (depuis `backend/`) : `npx prisma migrate dev --name <nom>` (CLI lit `backend/.env` → `DATABASE_URL="file:../db/oche.db"`). **Prisma 6 épinglé** (la v7 casse l'API CommonJS — ne pas upgrader sans migration dédiée). À chaque changement de schéma : migration + `npx prisma generate`.
- **Smoke-test backend** : `node backend/smokeTest.js` (36 assertions : auth, games/stats, social, blocage, feed, défis, classements, duel WebSocket+Elo, suppression de compte) — contre un serveur lancé sur **base temporaire** : `DATABASE_URL='file:../db/.smoke.db'` + `npx prisma migrate deploy` + `PORT=3102 SMOKE_PORT=3102`.
- **Typecheck** (à lancer après chaque lot de changements) : `npx tsc --noEmit -p tsconfig.json`. Sur Windows utiliser l'outil PowerShell.

## ⚠️ Règles critiques (à respecter absolument)
- **Sécurité BDD** : la base réelle est **`backend/db/oche.db` (SQLite/Prisma)**. Avant tout smoke-test, lancer le serveur sur un **PORT temporaire** avec **`DATABASE_URL` pointant une base jetable** (`file:../db/.smoke.db` + `migrate deploy`) — ne jamais tester contre la vraie base. **Ne jamais détruire les données utilisateur.** Le sandbox bloque `Remove-Item` sur certains chemins → fallback `Move-Item`. L'ancien `oche-db.json` + sauvegardes `*.prereset-*`/`*.prepurge-*` restent dans `backend/db/` (source d'import : `node backend/importJson.js [chemin] [--force]`).
- **Compte à préserver** : `@admin` uniquement — base **remise à zéro le 11/06/2026** pour le déploiement (sauvegardes complètes `*.prereset-*` / `*.prepurge-*` dans `backend/db/`). Les données de démo sont taguées `demo: true` (purgeables, `node backend/purgeDemo.js`).
- `JWT_SECRET` = défaut dev `'oche-dev-secret-change-me'` (à changer en prod). `.env` est gitignoré — ne jamais committer de secret.
- Commits/push **uniquement si demandé**.

## Stack & archi
- **Client** : expo-router (Stack racine dans `app/_layout.tsx`, onglets sous `app/tabs/`). État : **Zustand** (`useGameStore`, `useAuthStore`, `useThemeStore`, `useTournamentStore`, `usePracticeStore` [persist+AsyncStorage]). Data serveur : **@tanstack/react-query** (singleton `services/queryClient.ts`).
- **Backend** : Node/Express + **Prisma 6 / SQLite** (`backend/db/oche.db`, schéma `backend/prisma/schema.prisma`, prêt à passer Postgres). Accès via `backend/db/prisma.js` : singleton + helpers `jarr/jobj/jstr` (SQLite n'a pas de Json natif → champs documents **sérialisés en String** : visits, heatmap, pushTokens, badges, opponents, likes, confirmedBy) + mappers `gameToApi`/`userToPublic` qui rendent les **formes historiques de l'API** (user_id, finished_at…) — le contrat client n'a pas bougé. `bcryptjs` + `jsonwebtoken`, middleware `requireAuth`. **WebSocket temps réel** dans `backend/realtime.js` (chemin `/ws`).
- Routes backend : `auth, users, games, feed, leaderboard, league, live, social, posts, challenges, notifications`. Agrégats stats dans `backend/stats.js` (`computeStats(gamesApi, userId)` — pur, sans accès BDD). Nouvelles routes : **PATCH/DELETE `/auth/me`** (édition nom/avatar/mdp, suppression de compte en cascade), **POST/DELETE `/social/block/:id`** + `GET /social/blocked` (bloqués exclus de la recherche, du feed et des défis), **GET `/users/:id/elo-history`** (points `EloPoint` écrits à chaque duel online classé, annulés si signalement).

Saisie : grille / cible SVG / numpad. **Bots 8 niveaux** (`hooks/botEngine.ts`, clés `avg30`→`avg100` = moyenne 3-fléchettes visée ; X01 simulé **fléchette par fléchette** avec routes de checkout, doubles ratés et busts réalistes ; anciennes clés `easy/medium/hard/pro` toujours acceptées — tournois persistés ; calibration : `npx esbuild hooks/botEngine.ts --bundle --format=esm --outfile=scripts/.botEngine.bundle.mjs` puis `node scripts/simBots.mjs`). **Undo** d'une volée validée (snapshots, `undoLastVisit`). **Qui commence ?** (`components/StartDecider.tsx`) : overlay avant le 1er tir en multijoueur local — tirage au sort animé ou bull manuel ; flag store `starterChosen` (reset par chaque `init*`) + action `chooseStarter`. **Score X01 en direct** : le restant du joueur actif descend à chaque fléchette en modes par-fléchette (`liveRemaining` dans scoring).
- **Tournois** (élimination + poules, podium, 3e place, undo, équipes), **Entraînement** (drills + records persistés par compte, barre de progression + record à battre ; classiques inclus : Bob's 27, **Catch 40**, **Le 121**, Highscore, tours doubles/horloge, checkouts).
- **Sons d'ambiance** (`services/soundService.ts`, expo-audio) : 180/Shanghai, checkout de leg, bust, victoire — en local **et** en online. Assets WAV synthétisés par `node scripts/genSounds.mjs` → `assets/sounds/`. Réglage On/Off dans Profil (clé device `oche.sounds`). Pas de voix de caller pour l'instant (TTS jugé trop artificiel — piste : samples enregistrés).
- **Social** : feed/posts/likes/commentaires, follow, **défis** asynchrones, **badges** (cliquables → fiche détail avec progression vers l'objectif), notifications + cloche.
- **Chat temps réel** (`app/messages.tsx` liste, `app/chat/[id].tsx` fil, `app/new-group.tsx`) : DM 1v1 + **groupes**, via le WebSocket global (`realtime.js` `pushToUsers` → events `chat_message`/`chat_read`/`chat_member_added` relayés ; `LiveInviteListener` invalide les caches React Query `chat-conversations`/`chat-messages`/`chat-unread`). Non-lus (`lastReadAt` par membre) + badge sur l'onglet Online + push best-effort. **Lancer un 1v1 depuis le chat** : bouton 🎯 → `online-match?host=1&conv=<id>` crée un salon privé et poste un message `kind=match_invite` (code dans `meta`) ; les 2 premiers qui tapent « Rejoindre » jouent. Bloqués exclus des DM. Bouton « 💬 Message » sur le profil joueur. **Pas encore** : tournoi de groupe + partage des résultats (lié au multijoueur 3+).
- **Online temps réel** (WebSocket, `app/online-match.tsx` + `services/liveSocket.ts` singleton connecté à l'auth) : partie rapide, salon privé, **format au choix** (301/501/701, legs, finish), **défi live depuis le profil** (présence), **chat**, **réactions emoji**, **mode spectateur** + liste « en direct », **présence amis en ligne**.
- **Online multijoueur 3-6** (`realtime.js`, room.maxPlayers 2-6) : salle avec **lobby** (event `lobby`, démarrage manuel par l'hôte = players[0] via msg `start`, ou auto quand pleine), rotation de tour circulaire `(turn+1)%n`, départ tournant à chaque leg, fin à `legsToWin`. UI : sélecteur 2-6 joueurs à la création, écran lobby, scoreboard compact N joueurs. **Elo seulement en 1v1** (n===2) — les parties 3+ comptent en stats/historique mais pas l'Elo. ⚠️ Un départ termine la salle pour tous (pas de reprise à chaud). **Lancer un match de groupe** depuis le chat : 🎯 → `online-match?host=1&conv=<id>` (idle → choix format+joueurs → create) → poste l'invitation `match_invite` dans la conv → membres rejoignent (lobby) → hôte lance. **Fix au passage** : course WS (handler de message branché synchrone + file d'attente jusqu'à l'auth, sinon un message envoyé juste après connexion était perdu).
- **Classement Elo** des duels online (K=40 provisoire/24, défaut 1000) — métrique par défaut du classement Online. **Anti-triche** : moyenne « surhumaine » (>140) → 0 Elo + `suspect` ; **signalement** en fin de match → Elo annulé + `flags` ; joueurs flaggés (≥3) marqués ⚠.
- **Stats avancées** : moyenne, first-9, checkout %, 180s, doubles, heatmap, **tranches 60/100/140/180**, fléchettes/leg, meilleur leg, par type de jeu, **courbe Elo/moyenne**.
- **Replays** : chaque partie stocke le **journal des volées** (`visits`) → écran `app/game/[id].tsx` (volée par volée + courbe). Parties online taguées `online` (badge 🔴).
- **Mode invité / hors-ligne** : « Jouer en invité » sur l'auth ; jeu local complet sans compte ; onglets compte (Stats/Online/Feed/Profil) affichent `ConnectPrompt` ; **étanchéité** : `queryClient.clear()` à chaque transition d'auth, cloche masquée hors-compte, records d'entraînement **compartimentés par compte** (`u<id>`/`guest`).
- **Géoloc** (classement régional/Europe/pays), **push** (prêt pour build dev ; Expo Go ne reçoit pas le push).

## Modèle de données (= `backend/prisma/schema.prisma`)
- `User` : id, name, username (unique), passwordHash, createdAt, country/countryCode/region/city, **avatarUrl**, elo (1000), eloGames, flags, demo, pushTokens/badges (String JSON). Relations cascade : games, posts, comments, follows, challenges, notifications, blocks, eloHistory.
- `Game` : id, userId, gameType, matchWon, legsWon/legsPlayed, opponents/opponentIds (String JSON), dartsThrown, avg, total180s, highestCheckout, score, heatmap (String JSON), checkout*/doubles/first9*, startScore, **visits (String JSON — replays ; futurs `coords` caméra dedans, sans changement de schéma)**, online, suspect, reported, confirmed, confirmedBy (String JSON), demo, finishedAt.
- `Follow` {followerId+followingId (pk), createdAt}, `Post` {likes String JSON}, `Comment`, `Challenge` {fromId,toId,gameType,legsToWin,message,status}, `Notification` {userId, type, actorId/postId/gameId/challengeId sans FK (null-checkés), badge, read}, **`Block`** {blockerId+blockedId (pk)}, **`EloPoint`** {userId, elo, gameId?, createdAt}.
- **Chat** : `Conversation` {isGroup, name?, createdById}, `ConversationMember` {conversationId+userId (pk), lastReadAt — base des non-lus}, `ChatMessage` {conversationId, senderId, text, kind 'text'|'match_invite', meta String JSON (ex. {code})}. Routes `backend/routes/chat.js` (montées sur `/chat`).
- L'API expose toujours les formes historiques (`user_id`, `finished_at`, tableaux parsés) via `gameToApi`/`userToPublic` — ne pas renvoyer d'objets Prisma bruts.

## App web (même codebase)
- **Servie par le backend** (`backend/server.js` → `express.static('backend/web')` + fallback SPA) sur le domaine Railway : `https://hoche-backend-production.up.railway.app`. iPhone/PC : ouvrir l'URL dans le navigateur.
- **Build web** : **`npm run web:build`** = `expo export -p web --output-dir backend/web` **puis** `node scripts/patchWeb.mjs`. ⚠️ Le patch est **obligatoire** : Expo SDK 54 émet `import.meta.env` (middleware devtools de zustand) mais charge le bundle en `<script>` classique → `SyntaxError: Cannot use 'import.meta'` → **écran blanc**. `patchWeb.mjs` remplace `import.meta` par `({})` dans les bundles. Toujours rebuild via `web:build` (jamais `expo export` seul).
- **`backend/web/` est versionné** (committé) — `railway up` respecte `.gitignore`, donc l'ignorer le retire du déploiement (déjà causé un « Cannot GET / »). Rebuild + commit avant `railway up`.
- Déploiement web = même `railway up --detach --service hoche-backend` que le backend (le web part avec).
